    5. Create backend/src/services/sqlValidationService.js with these exports:

       - `validateFlinkSql(sql)`: Uses `new FlinkSQL()` from dt-sql-parser. Returns `{ valid: boolean, errors: Array<{line, col, message}> }`. Uses `flinkParser.validate(sql)` for syntax check.

       - `splitStatements(sql)`: Uses `flinkParser.splitSQLByStatement(sql)` for grammar-aware splitting (NOT regex split on semicolons).

       - `extractFieldsFromSQL(sql)`: Regex extraction of field names from SELECT, WHERE, GROUP BY, ORDER BY, JOIN ON clauses. MUST handle table-aliased fields: strip `tablealias.` prefix from patterns like `o.order_id` to extract `order_id`. Return array of unique field name strings.

       - `catalogCheck(sql, schemasByTopic)`: Takes SQL string and a map of `{ topicName: { fields: [{name, type}] } }`. Builds `knownFields` Set from all schema fields. Calls `extractFieldsFromSQL(sql)`. Returns array of issue strings for:
         (a) Unknown fields: field not in knownFields set
         (b) STRING field in numeric context: if `total_amount` appears in SUM/AVG/MIN/MAX/arithmetic without TRY_CAST nearby — issue: "total_amount is STRING type — use TRY_CAST(total_amount AS DECIMAL(10,2))"
         (c) BIGINT timestamp without conversion: if `created_at` or `event_time` of type BIGINT appears without TO_TIMESTAMP_LTZ nearby — issue: "created_at is BIGINT epoch millis — use TO_TIMESTAMP_LTZ(created_at, 3)"

       - `validateAndClassify(sql, schemasByTopic)`: Orchestrates the full pipeline:
         (a) Call validateFlinkSql(sql) — if errors, return `{ status: 'red', syntaxErrors, catalogIssues: [] }`
         (b) Call catalogCheck(sql, schemasByTopic) — if unknown fields, return `{ status: 'red', syntaxErrors: [], catalogIssues }`
         (c) If SQL contains TUMBLE/HOP/SESSION window functions, check if CREATE TABLE DDL has WATERMARK clause. If no watermark found, return `{ status: 'yellow', syntaxErrors: [], catalogIssues: [], warnings: ['Window function used but no WATERMARK declaration found in source DDL'] }`
         (d) Otherwise return `{ status: 'green', syntaxErrors: [], catalogIssues: [] }`

       Flink reserved words list to embed: `timestamp`, `time`, `user`, `order`, `value`, `row`, `interval`, `table`, `key`, `comment`. Use in catalog check to warn if these appear unquoted (without backticks).
