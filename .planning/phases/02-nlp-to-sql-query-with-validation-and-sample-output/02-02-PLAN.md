    2. Create backend/src/services/mockDataService.js:

       `parseMockRows(llmResponse)` — Takes the parsed JSON response object from Claude. Extracts `mockRows` array. Validates each row has keys matching `outputSchema` fields. Returns the array, or empty array if malformed.

       `generateFallbackMockRows(outputSchema, count = 5)` — Takes outputSchema array of `{field, type}`. Generates `count` rows with placeholder values:
       - STRING/VARCHAR: `"sample_" + index`
       - BIGINT/INT: `1000 + index`
       - DECIMAL: `(99.99 + index).toFixed(2)`
       - TIMESTAMP: `new Date().toISOString()`
       - BOOLEAN: `index % 2 === 0`
       Returns array of row objects.

    3. Create backend/src/services/llmService.js:

       `callClaude(systemPrompt, messages)` — Creates `new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY })`. Calls `anthropic.messages.create({ model: 'claude-sonnet-4-6', max_tokens: 4096, system: systemPrompt, messages })`. Returns `result.content[0].text`. IMPORTANT: API key comes from process.env only — never from client.

       `parseJsonResponse(text)` — Extracts JSON from Claude's response. Tries `JSON.parse(text)` first. If that fails, tries to extract JSON from markdown code fences (```json ... ``` or ``` ... ```). Returns parsed object or throws with descriptive error.

       `generateFlinkSQL(nlQuery, schemas, messageHistory = [])` — Main generation function:
       (a) Calls `buildSystemPrompt(schemas)` from prompts/systemPrompt.js
       (b) Builds messages array: if messageHistory is empty, `[{ role: 'user', content: nlQuery }]`. If messageHistory has entries, appends `{ role: 'user', content: nlQuery }` to existing history.
       (c) Calls `callClaude(systemPrompt, messages)`
       (d) Parses response with `parseJsonResponse(text)`
       (e) Returns `{ sql, outputSchema, mockRows, reasoning, rawResponse: text }`

       `generateWithSelfCorrection(nlQuery, schemas, messageHistory = [], maxRetries = 3)` — The locked-decision 3-retry loop:
       (a) Calls `generateFlinkSQL(nlQuery, schemas, messageHistory)` to get initial SQL
       (b) Imports `validateAndClassify` from sqlValidationService.js
       (c) Calls `validateAndClassify(result.sql, schemas)` to get `{ status, syntaxErrors, catalogIssues }`
       (d) If status is 'green' or 'yellow': return `{ ...result, validation: { status, attempts: 1 } }`
       (e) If status is 'red': build correction prompt with errors, append to messages:
           `[...currentMessages, { role: 'assistant', content: result.rawResponse }, { role: 'user', content: buildCorrectionPrompt(result.sql, [...syntaxErrors, ...catalogIssues]) }]`
       (f) Retry up to maxRetries times. Each retry re-generates and re-validates.
       (g) After maxRetries: return `{ ...lastResult, validation: { status: 'red', attempts: maxRetries, errors: lastErrors } }`

       `buildCorrectionPrompt(sql, errors)` — Returns string:
       ```
       The following Flink SQL has validation errors:

       ```sql
       {sql}
       ```

       Errors:
       {errors.map(e => `- ${e.message || e}`).join('\n')}

       Fix ALL errors and respond with the corrected JSON (same format as before).
       ```

       Session management: Export a `sessions` Map for conversation history. Key = sessionId (UUID from client). Value = { messages: [], lastAccess: Date.now() }. Add cleanup function that removes sessions older than 1 hour.
