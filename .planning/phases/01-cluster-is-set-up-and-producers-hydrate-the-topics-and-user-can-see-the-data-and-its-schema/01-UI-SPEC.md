## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline chip padding, border accents |
| sm | 8px | List item internal padding (vertical), tag spacing |
| md | 16px | Panel internal padding, sidebar item padding (horizontal and vertical) |
| lg | 24px | Section header bottom margin, panel-to-panel gap |
| xl | 32px | Header vertical padding budget |
| 2xl | 48px | Not used in Phase 1 |
| 3xl | 64px | Not used in Phase 1 |

Exceptions:
- Header: 56px total height (14px top + 14px bottom + 28px content) — aligns to 4px grid

Touch target verification — sidebar items: `py-4` (16px top + 16px bottom) + 14px font line = 46px — exceeds 44px minimum.

## Color

Dark theme. All values map to Tailwind slate/gray palette from flinkenstein reference.

| Role | Tailwind Class | Approx Hex | Usage |
|------|---------------|------------|-------|
| Dominant (60%) | `bg-gray-900` | `#111827` | App background, raw JSON pane background, message card backgrounds |
| Secondary (30%) | `bg-gray-800` | `#1f2937` | Sidebar background, detail panel background, card/panel surfaces |
| Accent primary (8%) | `text-cyan-400` / `bg-cyan-400` | `#22d3ee` | Selected topic highlight text, schema topic name, WebSocket live badge border, active tab underline, left-border accent on selected message rows |
| Accent secondary (2%) | `text-purple-400` | `#c084fc` | App title secondary word ("flink**enstein**"), domain badge for Fraud detection topics |
| Destructive | `text-red-500` / `bg-red-900/20` | `#ef4444` | Error states only — failed connection, schema load error |
| Success semantic | `text-green-400` | `#4ade80` | Live connection badge, WebSocket connected state |
| Warning semantic | `text-yellow-400` | `#facc15` | Producers disconnected, stale data warning |

Accent reserved for:
- Selected topic name text in sidebar (`text-cyan-400`)
- Selected topic name in panel heading (`text-cyan-400`)
- Active tab underline indicator
- Left-border accent on the currently-streaming message row (`border-l-2 border-cyan-400`)
- Live/streaming status badge border
- Domain badge for Retail topics

Surface hierarchy:
- `bg-gray-900` (page) > `bg-gray-800` (panels) > `bg-gray-700` (hover state) > `bg-gray-700` (selected sidebar item uses `bg-purple-600` per flinkenstein pattern)

Selected sidebar item: `bg-purple-600 text-white font-semibold` (carried from flinkenstein — distinguishes selection from hover)

## Interaction Contract

### Topic Selection
- Click a topic in sidebar: immediately highlights topic (`bg-purple-600`), switches detail panel to Schema tab, triggers schema fetch and first-batch message fetch
- No loading skeleton — use spinner states within the panel instead

### Tab Switching (Schema / Messages)
- Click tab: instant render from cached data, no fetch on tab switch (data already in state from topic selection)

### Message View Toggle (Table / Raw JSON)
- Click toggle: instant re-render of same message data in new format, no network request
- Default on topic select: Table view

### WebSocket Live Streaming
- On topic select: WebSocket connection established to backend for that topic
- Messages prepended to list (newest first, max 50 messages buffered in UI)
- No manual "refresh" needed; live badge shows connected state
- On topic deselect or topic switch: previous WebSocket connection closed

### Sidebar Refresh
- Click refresh icon: invalidates topic list query, re-fetches from backend
- Spinner replaces refresh icon during fetch

### Scroll
- Sidebar: vertical scroll when topics exceed viewport
- Message list: vertical scroll within panel, max-height fills viewport minus header + tab bar
- Wide tables: horizontal scroll on message table and schema table, contained within panel

### Loading States
- Topic list loading: single centered spinner in sidebar area
- Schema loading: centered spinner in detail panel with text "Loading schema..."
- Message loading: centered spinner with text "Connecting to live stream..."

### Error States
- All errors: inline within the relevant panel zone, never full-page errors
- Schema error: red callout in schema panel area
- WebSocket disconnect: live badge changes to "Disconnected" in amber, messages freeze but remain visible

## Accessibility

- All interactive elements have visible focus rings: `focus:outline-none focus:ring-2 focus:ring-cyan-400 focus:ring-offset-2 focus:ring-offset-gray-900`
- Sidebar topic list items: `role="listbox"` / `role="option"` with `aria-selected`
- Tab bar: `role="tablist"` / `role="tab"` with `aria-selected` and `aria-controls`
- Live badge: `aria-live="polite"` for connection state changes
- Message list: `aria-live="polite" aria-atomic="false"` for new message prepend
- Spinner elements: `aria-label="Loading"` with `role="status"`
- Refresh icon button: `aria-label="Refresh topic list"` (icon-only button, no visible text label)
- Minimum touch target: 44px height on sidebar items (achieved via `py-4` + 14px font line = 46px — passes)
- Color contrast: all text on dark surfaces meets WCAG AA (gray-200 on gray-800 = 7.5:1; cyan-400 on gray-800 = 4.8:1)
